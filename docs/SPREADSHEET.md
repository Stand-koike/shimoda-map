# スプレッドシートマスタ仕様

本ドキュメントは [`map/index.html`](../map/index.html) の `CONFIG.COLS`（**0 始まりの列インデックス**）と一致させる**正本**です。ヘッダー行の**列順**は [samples/spreadsheet-headers.csv](./samples/spreadsheet-headers.csv) と同一です。

---

## 1. 公開設定

- アプリは **Google Visualization API（JSONP）** でシートを読みます。
- スプレッドシートは **「リンクを知っている全員が閲覧可」** など、匿名で `gviz/tq` が読める状態にする必要があります（編集権限はマスタ管理者・AppSheet に限定）。
- **機微情報・個人の特定に直結する情報**は、公開前提で載せる範囲を運用で決めてください。

---

### 画像 URL の登録（手動・AppSheet の両方）

マップは **`image_url` / `image_url_2` / `image_url_3`** に入った文字列を、ブラウザで `<img>` 表示しやすい形へ自動変換します（[`normalizeImageUrl`](../map/index.html)）。次のどちらでも運用できます。

#### 方法1: スプレッドシートに手動で貼る（管理者・一括登録）

1. **Google Drive** に画像をアップロードする。
2. ファイルを右クリック → **共有** → **リンクを知っている全員が閲覧可**（少なくとも「閲覧者」）にする。  
   ※ 閲覧できない URL だとマップ上も画像は表示されません。
3. **リンクをコピー**し、`image_url` などのセルに貼り付ける。  
   - `https://drive.google.com/file/d/FILE_ID/view?usp=sharing`  
   - `https://drive.google.com/open?id=FILE_ID`  
   など、Drive が出す形式はそのままで構いません。

#### 方法2: AppSheet から店舗ユーザーがアップロード

1. AppSheet で **Image**（またはファイル）型の列を用意し、**データソースの列**をスプレッドシートの `image_url`（および必要なら `image_url_2` / `image_url_3`）と対応させる。
2. AppSheet の **Folder** 設定で、画像が **Google Drive 上のフォルダ**に保存されるようにする（既定の動作で問題ないことが多い）。
3. ユーザーがアプリから写真を選ぶと、シートには **Drive の URL** や **`export=download`** 付きの `uc?` URL などが入ることがありますが、マップ側で **`uc?export=view&id=`** 形式へ揃えて表示します。
4. 画像が表示されない場合は、(1) そのファイルの共有が「閲覧可」か、(2) シートに入っている文字列が `https://` で始まる完全な URL かを確認してください。

**共通の注意**: 公開マップに載せる画像のみを格納するフォルダ運用にし、個人情報が写り込まないよう店舗側ガイドを用意すると安全です。

### `hidden` 列の動作（実装の注意）

- 行の `hidden` が **boolean の `false`** または **文字列 `'FALSE'`** のとき、その行は**表示しない**（非表示扱い）。
- これ以外（空、TRUE、TRUE 以外の文字列）は**表示**されます。運用では「非表示にしたい行だけ FALSE を入れる」形に揃えると安全です。

---

## 3. AppSheet 運用メモ

- **マスタ**はスプレッドシート 1 本。AppSheet は「データソース」として同一スプレッドシートを参照する想定。
- 店舗ユーザーは **自分の行のみ編集**できるよう、AppSheet 側で **行レベルセキュリティ** または **ユーザー別スライス**（`store_id` やメールと行の対応）を設定する。
- **`store_id`（列 J）** を各店舗に一意に付与すると、サポート・ディープリンク・将来のルート機能で扱いやすくなります。
- Web アプリの再取得は **ポーリング**（既定 30 秒）と **`?refresh=1`** による即時再取得に対応。AppSheet 保存後、数秒〜数十秒でシートに反映され、ポーリングで取り込まれます。

---

## 4. 既存シートからの移行

- 列 **J–K（9–10）** が空の既存データでも動作します（`store_id` 省略時は内部 ID）。
- 列 **V–Y（21–24）** を追加した場合、ヘッダー名は上表の推奨に合わせ、**列位置**を `COLS` と一致させてください。`index.html` の `CONFIG.COLS` を変更した場合は本ドキュメントと CSV も更新します。
